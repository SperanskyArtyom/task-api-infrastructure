# Cloud-Native Task API: Infrastructure & Deployment

Учебный проект по дисциплине «Архитектура распределённых приложений» (СибГУТИ, 2025). 
Демонстрация перехода от локальной разработки к масштабируемой инфраструктуре в Kubernetes с использованием GitOps-подхода.

## 🌟 Ключевые инженерные решения

### 1. Инфраструктура как код (IaC)
- **Helm & Helmfile:** Приложение полностью шаблонизировано. Использование `helmfile.yaml.gotmpl` позволяет декларативно управлять `dev` и `prod` окружениями, избегая дублирования конфигурации.
- **Kubernetes Native:** Описаны ресурсы `Deployment`, `Service`, `Ingress`. Настроены политики обновления (RollingUpdate).

### 2. Безопасность и отказоустойчивость
- **Hardened Containers:** В манифестах реализован запрет на запуск от root-пользователя (`runAsNonRoot: true`) и сброс всех потенциально опасных capabilities ядра (`drop: ["ALL"]`).
- **Health Checks:** Настроены Liveness и Readiness пробы, интегрированные с Flask-эндпоинтом `/health`, что гарантирует автоматическое восстановление сервиса при сбоях.
- **Graceful Shutdown:** В коде приложения реализована обработка сигналов `SIGTERM` и `SIGINT` для корректного завершения соединений при рестарте подов.

### 3. Автоматизация (CI/CD)
- **Multi-arch Build:** Настроен `docker buildx` для сборки образов под архитектуры `amd64` и `arm64`.
- **GitOps Flow:** Реализован пайплайн с этапами:
  - `Build`: Сборка и пуш в приватный реестр.
  - `Diff`: Проверка изменений в инфраструктуре перед деплоем (`helmfile diff`).
  - `Deploy`: Ручной (manual) или автоматический деплой в зависимости от окружения.

## 🛠 Технологический стек
- **Backend:** Python (Flask)
- **CI/CD:** GitLab CI
- **Orchestration:** Kubernetes, Helm, Helmfile
- **Server:** Traefik Ingress

## 📂 Структура проекта
```
├── app.py               # REST API на Flask с поддержкой Graceful Shutdown
├── Dockerfile           # Оптимизированный multi-stage Dockerfile
├── helm/                # Helm-чарт для развертывания приложения
│   ├── templates/       # Шаблоны k8s манифестов
│   └── values.yaml      # Базовые конфигурации сервиса
├── helmfile.yaml.gotmpl # Описание релизов для сред Dev/Prod
├── .gitlab-ci.yml       # Описание стадий build, diff и deploy
└── kubernetes/          # Базовые K8s-манифесты (legacy/reference)
```

## 🚀 Как запустить локально
### 1. Сборка образа
```bash
docker build -t task-api:latest .
```
2. Развертывание через Helmfile (требуется k8s кластер либо minikube)
```bash
# Просмотр ожидаемых изменений
helmfile -e default diff

# Применение конфигурации
helmfile -e default apply
```
